
# Разработка программно-аналитического комплекса автоматизации расчётных процедур промышленной безопасности

## 1. Проблема

При разработке паспортов безопасности объектов ТЭК и иных опасных производственных объектов (ОПО) инженер обязан:

* рассчитывать параметры взрывов ТВС
* определять зоны поражения
* оценивать вероятность разрушений
* рассчитывать людские и экономические потери
* формировать регламентированную документацию

Расчёты выполняются вручную или в разрозненных Excel-файлах на основе:

* ГОСТ Р 12.3.047-2012
* Приказа Ростехнадзора №412
* Методик МЧС
* Методик Минэнерго

Это приводит к:

* высокой трудоёмкости
* риску ошибок
* отсутствию единого вычислительного ядра
* сложности масштабирования расчётов

---

## 2. Цель проекта

Создание программно-аналитической платформы, автоматизирующей расчётные процедуры оценки последствий аварийных сценариев и формирование регламентированной документации промышленной безопасности.

---

## 3. Технологическое решение

Разрабатывается вычислительное ядро, реализующее:

### Модуль 1 — Расчёт массы выброса

* определение расхода газа
* расчёт объёма ГВС
* определение энергозапаса смеси

### Модуль 2 — Определение режима взрывного превращения

* выбор диапазона дефлаграции/детонации
* учёт загромождённости пространства
* выбор скоростных параметров

### Модуль 3 — Расчёт параметров ударной волны

* безразмерные величины Px и Ix
* расчёт ΔP и импульса
* определение зон разрушений

### Модуль 4 — Пробит-анализ

* оценка вероятности разрушений
* оценка вероятности поражения людей

### Модуль 5 — Расчёт социально-экономического ущерба

* ущерб населению
* ущерб окружающей среде
* ущерб государству

### Модуль 6 — Автоматическая генерация отчёта

* формирование Word-документа
* структурированная подстановка результатов
* готовность к передаче в надзорные органы

│  │
│  ├─ report/                    # Формирование отчётных документов
│  │  ├─ templates/
│  │  │  ├─ template.docx        # Word-шаблон отчёта (docxtpl)
│  │  │  └─ __init__.py
│  │  ├─ word_builder.py         # Сбор контекста + генерация Word отчёта
│  │  └─ __init__.py
│  │
│  ├─ ui/                        # UI-слой (вариант интерфейса)
│  │  ├─ main_window.py
│  │  └─ __init__.py
│  │
│  ├─ ui_tk/                      # UI-слой на Tkinter (альтернативный интерфейс)
│  │  ├─ main_window_tk.py
│  │  └─ __init__.py
│  │
│  ├─ main.py                     # Точка входа приложения
│  └─ __init__.py
│
├─ out/                           # Артефакты генерации (формируются автоматически)

Логика работы:

Входные данные →
Формализация сценария →
Алгоритмический расчёт →
Численное определение параметров →
Формирование документа

## 5. Научно-техническая новизна

1. Формализация нормативных методик в виде алгоритмического вычислительного ядра.
2. Интеграция нескольких методических документов в единую модель.
3. Реализация безразмерного подхода к расчёту ударных волн.
4. Численная реализация расчёта зон поражения.
5. Автоматизация пробит-анализа вероятностей.

В отличие от Excel-решений, система:

* имеет параметрическую архитектуру
* допускает масштабирование
* обеспечивает воспроизводимость расчётов
* формирует отчёт без ручной корректировки
